# SECURITY POLICY

## 1. Overview
ZERO is designed for high-security, regulated, and air-gapped environments. We prioritize data sovereignty and minimal attack surface.

## 2. Infrastructure
- **Self-Hosting**: Users deploy ZERO containers on their own hardware.
- **No Telemetry**: By default, no usage data is sent to ZERO Inc.
- **No Remote Access**: ZERO Inc. cannot access user data or keys once deployed.

## 3. Vulnerability Reporting
If you discover a security vulnerability, please report it privately to `security@zero-s.tech`.
DO NOT open public GitHub issues for security vulnerabilities.

## 4. Compliance
Designed to align with:
- ISO/IEC 27001
- GDPR
- DPDP Act (India)
